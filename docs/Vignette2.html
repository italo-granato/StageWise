Vignette 2: Single trait analysis at correlated locations

This vignette builds on the information in Vignette 1, which covered analysis at a single location. This vignette illustrates how to model the covariance between multiple locations, using a dataset of potato yield trials across 9 locations and 6 years and a population size of 336 genotypes (Schmitz Carley et al. 2019).

When the data frame of BLUEs passed to Stage2 has a column labeled ‘loc’, genotype x location effects are included using a separable covariance structure. When marker data are included, the genetic covariance between locations for the additive effects follows a 2nd order factor-analytic (FA2) model. Based on experience with a few datasets, attempts to also model covariance between residual genetic effect were unsuccessful, so the correlation is constrained at 1 in the current implementation. If marker data are not included, the genetic covariance between locations follows the FA2 model. For large datasets with many locations (such as this one), I recommend first analyzing it without marker data, as the computation proceeds more quickly.

As in the single location analysis, running the summary command on the ‘vars’ output shows the partitioning of variance, but now a g x loc effect is also present. Although g x year effects could also be estimated, they are typically small and, unlike g x loc, have little practical importance; this is why they are not modeled explicitly in Stage2.

The summary command also returns the correlation matrix between the locations. One way to visualize this information is with a dendrogram, which shows that the TX site, and to a lesser extent the FL and CA sites, are different than the other six.

plot(hclust(as.dist(1-summary(ans2a$vars)[[2]])),
     hang = -1,xlab="")

The FA2 model should have enough complexity for a set of correlated locations used by a single breeding program, but it may be inadequate when analyzing disparate locations, such as in this dataset. Stage2 returns a uniplot (Cullis et al. 2010), which displays the locations against a unit circle (radius=1) to visualize model sufficiency and correlation structure.

ans2a$uniplot

The squared radius for each location is the proportion of genetic variance explained by the latent factors. With the exception of TX and FL, the FA2 model appears to provide a good representation of the covariance structure. The cosine of the angle between locations equals the correlation due to the latent factors (recall cos(0) = 1). Focusing on the 6 highly correlated locations-WI,MI,NC,OR,NY,MO-we will now add the marker data to Stage2 to estimate the additive correlation and predict breeding values.

locs <- c("WI","MI","OR","NY","NC","MO")
blues <- ans1$blue[ans1$blue$loc %in% locs,]
tmp <- sapply(strsplit(names(ans1$vcov),split="_"),"[[",1)
vcov <- ans1$vcov[tmp %in% locs]
ans2b <- Stage2(data=blues,vcov=vcov,geno=geno)
summary(ans2b$vars)

Compared to the single location analysis in Vignette 1, an additional argument is needed for the blup function to specify the index weights, which are interpreted as the relative weight for each location after standardization to unit variance. The following code compares the reliability for BV predictions in WI vs. NC.

prep1 <- blup_prep(data=blues,vcov=vcov,geno=geno,vars=ans2b$vars)

WI.index <- c(WI=1, NC=0, OR=0, NY=0, MO=0, MI=0)  
NC.index <- c(WI=0, NC=1, OR=0, NY=0, MO=0, MI=0)  

WI.pred <- blup(data=prep1,geno=geno,index.weights=WI.index,what="id")
NC.pred <- blup(data=prep1,geno=geno,index.weights=NC.index,what="id")

pred <- merge(NC.pred[,c("id","BV","BV.r2")],WI.pred[,c("id","BV","BV.r2")],by="id")
colnames(pred) <- c("id","BV.NC","r2.NC","BV.WI","r2.WI")

ggplot(pred,aes(x=r2.NC,y=r2.WI)) + geom_point() + coord_fixed(ratio=1) + xlim(0.3,0.8) + ylim(0.3,0.8) + geom_line(data=data.frame(x=c(0.3,0.8),y=c(0.3,0.8)),mapping=aes(x=x,y=y),linetype=2) + theme_bw() + 
  xlab("NC") + ylab("WI") + ggtitle("BV Reliability")

The above figure shows that breeding values were predicted with higher reliability in WI than NC, which is to be expected from the high correlation between WI and the other four sites (OR,MO,MI,NY) compared to NC.

As a pedagogical aside, one should avoid estimating genetic correlation via the sample correlation between GEBVs. This is because the correlation between BLUPs is an upwardly biased estimator of the true correlation. As shown above, the REML estimate of the genetic correlation between WI and NC was 0.67, while the correlation between the BLUPs is 0.74:

cor(pred$BV.NC, pred$BV.WI)  
## [1] 0.7400824

The mask argument for blup_prep can be used to mask individuals at one or more locations, to explore the accuracy of prediction into new environments. Whereas the previous analysis determined the reliability of GEBVs in WI when WI phenotypes are available, the next analysis excludes WI phenotypes:

WI.env <- blues$env[blues$loc=="WI"]

prep2 <- blup_prep(data=blues,vcov=vcov,geno=geno,vars=ans2b$vars,
                   mask=data.frame(env=WI.env))

pred2 <- blup(data=prep2,geno=geno,index.weights=WI.index,what="id")

plot.data <- merge(pred2[,c("id","BV.r2")],WI.pred[,c("id","BV.r2")],by="id")
colnames(plot.data) <- c("id","without.WI.pheno","with.WI.pheno")

ggplot(plot.data,aes(x=without.WI.pheno,y=with.WI.pheno)) + geom_point() + coord_fixed(ratio=1) + xlim(0.3,0.8) + ylim(0.3,0.8) + geom_line(data=data.frame(x=c(0.3,0.8),y=c(0.3,0.8)),mapping=aes(x=x,y=y),linetype=2) + theme_bw() + 
  xlab("Without WI phenotypes") + ylab("With WI phenotypes") + ggtitle("WI BV Reliability")

As expected, the reliability was higher with WI phenotypes.
